事务

什么是事务

  • 一件事情有n个组成单元 要不这n个组成单元同时成功 要不n个单元就同时失败，就是将n个组成单元放到一个事务中

四大特性ACID

原子性（Atomicity） 是指事务是一个不可分割的工作单位，事务中的操作要么都发生，要么都不发生
一致性（Consistency） 一个事务中，事务前后数据的完整性必须保持一致
隔离性（Isolation） 事务的隔离性是指多个用户并发访问数据库时，一个用户的事务不能被其它用户的事务所干扰，多个并发事务之间数据要相互隔离
持久性（Durability） 是指一个事务一旦被提交，它对数据库中数据的改变就是永久性的，接下来即使数据库发生故障也不应该对其有任何影响

并发访问问题

脏读 要求B事务要读取A事务提交的数据，但是B事务读取到了A事务尚未提交的数据；若A事务回滚，则B事务读取的数据就是临时且无效的
不可重复读 A事务读取了一个字段，然后B事务更新了该字段，此时A事务再一次读取该字段，两次读取的值不一致
幻读/虚读 A事务从表中读取了一个字段，然后B事务在该表中插入了一些新的记录，此时A事务再次读取该表，两次读取的数据量不一致

事务的隔离级别

  • 一个事务与其他事务隔离的程度称为隔离级别
read uncommitted 读取尚未提交的数据 ：哪个问题都不能解决
read committed 读取已经提交的数据 ：可以解决脏读 oracle默认的
repeatable read 重读读取：可以解决脏读 和 不可重复读 mysql默认的，在这个事务持续期间，禁止其他事务对这个字段进行更新
serializable 串行化：可以解决 脏读 不可重复读 和 虚读 相当于锁表，在这个事务持续期间，禁止其他事务对该表执行插入、更新和删除操作

隔离级别的性能

  • read uncommitted>read committed>repeatable read>serialazable

隔离级别的安全性

  • read uncommitted<read committed<repeatable read<serialazable

默认事务

  • 一条sql语句就是一个事务 默认就开启事务并提交事务

手动事务

  1. 显式开启一个事务
  2. 事务提交：commit代表从开启事务到事务提交中间的所有的sql都认为有效，此时才更新数据库
  3. 事务的回滚：rollback 代表事务的回滚 从开启事务到事务回滚 中间的所有的sql操作都认为无效，数据库没有被更新

存储引擎

类型

InnoDB：支持事务，不支持全文本搜索**（MySQL默认）**

MyISAM：性能极高，不支持事务，支持全文本搜索

Memory：功能上等同于MyISAM，存在于内存中，速度快，适用于临时表
