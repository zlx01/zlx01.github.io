Maven学习笔记

发表于 2020-05-17 更新于 2020-05-18 分类于 maven 阅读次数：
本文字数： 28k 阅读时长 ≈ 25 分钟

约定优于配置，Convention Over Configuration

什么是Maven

  • 官网：https://maven.apache.org/
  • 一个构建工具
  • 一个依赖管理工具
  • 一个项目信息管理工具

什么是构建

  • 编译、运行单元测试、生成文档、打包和部署等工作

构建工具

  • Make
  • Ant，Another Neat Tool

安装Maven

  • 以Windows为例
  • 必须先安装JDK
  • 下载地址：https://maven.apache.org/download.cgi，选择.zip

  • 解压到指定目录

  • 设置环境变量

  • 测试

  • 其他系统安装指引：https://maven.apache.org/install.html

仓库

中央仓库

  • https://mvnrepository.com/

Nexus私服

  • 参考博客

  • Nexus官网：https://www.sonatype.com/

  • 下载：https://www.sonatype.com/download-nexus-repo-oss

  • 解压到指定路径

  • 设置环境变量

  • 命令行运行nexus.exe /run，访问 http://localhost:8081/
  • 初始账号为admin，密码在本地文件中查看

  • 登录成功后要求重新设置密码，初始密码会失效，并且admin.password也会自动删除

  • 允许匿名访问

  • 仓库内容

本地仓库

  • 设置仓库镜像，提高下载速度。打开D:\program\apache-maven-3.6.2\conf\settings.xml，添加mirror
  • 修改本地仓库路径，默认放在C:\Users\Lee\.m2\repository，占用C盘容量。打开D:\program\apache-maven-3.6.2\conf\settings.xml，添加localRepository

IDEA集成Maven

  • IDEA本身集成了Maven

  • 一般都会切换成自己安装的版本

POM.xml

重要概念

  • Project Object Model，项目对象模型

  • GAV（groupId，artificatId，version）Maven中jar、war、pom的坐标、唯一标识

依赖范围scope

  • 区别：参考博客
scope 编译classpath 测试classpath 运行classpath 举例
compile（默认） Y U Y spring-core
test - Y - Junit
provided Y Y - servlet-api（运行时由容器提供）
runtime - Y Y JDBC驱动实现（编译时使用JDBC接口）
system Y Y - 本地的，Maven仓库之外的类库文件
import \ \ \ \

properties

  • 内置属性
  • POM属性
  • 自定义属性：供其他地方引用，可以集中管理依赖版本
  • Settings属性，如${settings.localRepository}
  • Java系统属性，如${user.home}，mvn help:system查看所有
  • 环境变量属性，如${env.JAVA_HOME}

profiles

  • 针对不同环境采用不同配置，例如开发和生产环境使用不同的数据库配置
  • springboot中有更方便的解决方案，不再赘述

模板参考

生命周期模型

参考官方文档

clean生命周期

  1. pre-clean
  2. clean：清理，例如target/
  3. post-clean

default生命周期

  1. validate
  2. initialize
  3. generate-sources
  4. process-sources
  5. generate-resources
  6. process-resources
  7. compile：编译项目代码
  8. process-classes
  9. generate-test-sources
  10. process-test-sources
  11. generate-test-resources
  12. process-test-resources
  13. test：编译运行测试代码
  14. prepare-package
  15. package：默认打包类型为jar
  16. pre-integration-test
  17. integration-test
  18. post-integration-test
  19. verify
  20. install：项目输出的jar会被保存到本地仓库中
  21. deploy：部署项目

site生命周期

  1. pre-site
  2. site
  3. post-site
  4. site-deploy

命令行与生命周期

  • mvn clean clean生命周期1、2阶段

  • mvn test default生命周期1~13阶段

  • mvn clean package clean生命周期1、2阶段+default生命周期1~15阶段

  • mvn clean install clean生命周期1、2阶段+default生命周期1~20阶段

  • mvn clean depoy clean生命周期1、2阶段+default生命周期所有阶段

插件

  • 生命周期是抽象的，具体任务是由插件代码完成的。
  • 生命周期和插件是相互绑定的

聚合与继承

聚合

  • 可以通过在一个打包方式为pom的Maven项目中声明任意数量的module元素来实现模块的聚合

继承

  • 子模块可以继承父模块的groupId，version，property，dependency，dependencyManagement等元素
  • 避免依赖重复
  • 集中统一管理依赖版本

版本管理

发布版和快照版

版本号约定

<主版本号>.<次版本号>.<增量版本号>-<里程碑版本>

主版本号：表示项目的重大架构变更

次版本：表示较大范围的功能增加和变化，及bug修复

增量版本：一般表示重大Bug的修复

里程碑版本：里程碑但非稳定版本，如3.0-alpha-1、3.0-alpha-2、3.0-beta-1
